io.js 光明的未来

由社区驱动的路线图

为了社区，依靠社区

io.js 是 node.js 的替代品，可互换，并且几乎兼容 npm 中的所有模块。

io.js 是一个社区驱动的开源项目，由 node.js 的主要贡献者发起，并且在最初的几个月内吸引了比 node.js 出现以来更多的活跃开发者。

从 io.js 能得到什么?

  1. 稳定性
  2. 透明性 (调试和追踪)
  3. 更好的 Streams
  4. 长期支持
  5. 拥抱标准
  6. 本地化

相关列表编辑自 roadmap 项目，欢迎贡献者!

稳定性策略

在各种语言中我们拥有最大的模块生态系统，我们不能打破他。

  • 不会移除 JS API。
  • 语义化版本。
    • 添加 API 会发布 minor 版本。
    • 如果 V8 的 C++ API 的变化能由 nan 处理，则发布 minor 版本，否则发布 major 版本。

来自路线图的 稳定性策略 章节，欢迎贡献者!

Channels

Release 任何被标记的版本都会编译
必须稳定且可用于生产环境。
Canary 每晚编译，包含 Canary V8 及其他新特性。
NG 可使用下一代 ES6/7 核心 API。


来自 路线图，欢迎贡献者!

NG (Next Generation)

  • 现代 JavaScript 应用的一个新平台。
  • 将会一直保持向下兼容。
  • 新的 API 将会以 ES6/7 为中心，但会兼容原来的模块和 API。
  • 将对现在平台的改善和对未来 API 的工作区分开。

NG 讨论还在持续进行中 NG 仓库

调试和追踪

应该在核心使用大量各种各样的调试和追踪工具，而不是提供高层级的抽象，如 domains。

  • 集成 trace_event (V8 和 Chrome 的新 trace API)。
  • 添加文档和布道这种调试生态系统。
  • 统一系统追踪终端 (DTrace，LTTng 等)。

由追踪工作组领导，欢迎贡献者!

Streams

  • Streams 工作组 已经接管了 readable-stream，它已经成为了 io.js 官方 stream 模块。
  • 解决所有现存的兼容问题。
  • 简化生成 stream，避免造成用户使用错误。
  • 探索 WHATWG Streams 接口，识别兼容性问题。
  • 提升 stream 性能。

由 Streams 工作组领导，欢迎贡献者!

长期支持

  • 新的发布线将会和稳定及所支持的依赖一起发布。
  • 只要有人一直积极的贡献，老版本依然会发布 patch 版本。
  • 一旦依赖库（如 v8）的维护者不再维护老版本的 io.js 中曾经引用过的版本，io.js 将会对此负责，继续维护。

从稳定性策略总结，欢迎贡献!

本地化

  • 减少引入 icu 的影响 (二进制和内存占用空间)。
  • 推进充满活力的本地化社区的发展
    • 自主性的语言社区 (到现在已经有 30+)。
    • 布道以增加成员人数 (到现在已经有 160+ 人)。
    • 优化工具，支持翻译站点，API 文档，安装器，PPT。
